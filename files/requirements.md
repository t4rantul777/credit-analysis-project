# Функциональные требования: Система обработки кредитных заявок

## 1. Назначение системы

Автоматизация процесса приёма, рассмотрения и принятия решений по кредитным заявкам клиентов.

---

## 2. Роли пользователей

| Роль | Описание |
|---|---|
| Клиент | Подаёт заявку, отслеживает статус |
| Аналитик | Рассматривает заявку, выносит решение |
| Администратор | Управляет пользователями и настройками |

---

## 3. Функциональные требования

### 3.1 Подача заявки
- Клиент заполняет форму: сумма, срок, цель кредита
- Система автоматически проверяет кредитный рейтинг клиента
- Заявке присваивается уникальный ID и статус `new`

### 3.2 Обработка заявки
- Заявка переходит в статус `review` и назначается аналитику
- Аналитик изучает данные клиента и выносит решение (`approve` / `reject`)
- При отказе обязательно указывается причина

### 3.3 Уведомления
- Клиент получает уведомление при смене статуса заявки
- Аналитик получает уведомление о новых назначенных заявках

### 3.4 Аналитика
- Дашборд с количеством заявок по статусам
- Отчёт по среднему времени обработки
- Топ причин отказа

---

## 4. REST API эндпоинты

| Метод | URL | Описание |
|---|---|---|
| POST | `/api/applications` | Создать заявку |
| GET | `/api/applications/{id}` | Получить статус заявки |
| PATCH | `/api/applications/{id}/status` | Обновить статус |
| POST | `/api/decisions` | Зафиксировать решение аналитика |
| GET | `/api/analytics/summary` | Получить сводную статистику |

### Пример запроса (создание заявки):
```json
POST /api/applications
{
  "client_id": 42,
  "amount": 150000.00,
  "purpose": "автокредит"
}
```

### Пример ответа:
```json
{
  "app_id": 1023,
  "status": "new",
  "created_at": "2025-06-01T10:30:00Z"
}
```

---

## 5. Нефункциональные требования

- Время ответа API — не более 500 мс
- Доступность системы — 99.5%
- Хранение данных — PostgreSQL
- Аутентификация — JWT-токены
