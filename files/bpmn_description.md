# BPMN: Процесс обработки кредитной заявки

## Описание процесса

Диаграмма описывает полный цикл обработки заявки — от подачи клиентом до финального решения.

## Участники (Lanes)

- **Клиент** — инициирует процесс
- **Система** — автоматические проверки
- **Аналитик** — ручная проверка и решение

## Шаги процесса

```
[Клиент подаёт заявку]
        ↓
[Система проверяет кредитный рейтинг]
        ↓
   < рейтинг < 300? >
    /            \
  Да             Нет
  ↓               ↓
[Авто-отказ]  [Заявка → статус "review"]
                  ↓
        [Аналитик изучает данные]
                  ↓
         < Одобрить? >
          /         \
        Да           Нет
        ↓             ↓
  [Одобрить]     [Отказать + причина]
        \             /
         ↓           ↓
    [Уведомить клиента]
                ↓
           [Конец]
```

## Ключевые бизнес-правила

1. Если кредитный рейтинг < 300 — автоматический отказ без ручной проверки
2. Если заявка висит в статусе `review` более 3 дней — эскалация руководителю
3. При отказе причина обязательна
4. Клиент уведомляется на каждом этапе смены статуса
